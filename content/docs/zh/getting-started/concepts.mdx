# 核心概念

在开始使用 Celoria 之前，了解以下核心概念将帮助您更好地理解系统。

## 组织架构

### 租户 (Tenant)

租户是 Celoria 中的最高级别组织单位，通常代表一个独立的企业或品牌。

- 每个租户拥有独立的数据空间
- 租户之间的数据完全隔离
- 一个租户可以管理多个门店

### 门店 (Store/Center)

门店是实际的经营场所，是日常运营的基本单位。

- 每个门店有独立的营业时间
- 员工分配到具体门店
- 预约、交易等都关联到门店

<Callout type="info">
如果您只有一家店，可以忽略多门店相关功能，系统会默认使用唯一的门店。
</Callout>

## 人员角色

### 员工 (Employee)

在系统中提供服务的工作人员。

| 属性 | 说明 |
|------|------|
| 技能 | 员工能提供的服务类型 |
| 排班 | 员工的工作时间安排 |
| 业绩 | 服务数量和销售金额统计 |

### 客人 (Guest)

来店消费的顾客。

| 属性 | 说明 |
|------|------|
| 档案 | 基本信息、联系方式 |
| 偏好 | 喜欢的服务、员工偏好 |
| 历史 | 消费记录、预约历史 |
| 会员 | 会员等级、积分余额 |

### 系统角色

| 角色 | 权限范围 | 典型职责 |
|------|----------|----------|
| 超级管理员 | 全部功能 | 系统配置、权限管理 |
| 管理员 | 大部分功能 | 日常管理、报表查看 |
| 经理 | 门店级别 | 排班、关账、团队管理 |
| 员工 | 基础功能 | 预约、收银、打卡 |

## 业务概念

### 服务 (Service)

门店提供的具体服务项目。

- **服务名称** - 如"经典美甲"、"手部护理"
- **服务时长** - 预计完成时间
- **服务价格** - 标准定价
- **服务分类** - 如"美甲"、"美睫"、"护理"

### 预约 (Appointment)

客人预订的服务时间安排。

**预约状态流转：**

```
已预约 → 已签到 → 服务中 → 已完成 → 已结账
                     ↘
                    已取消
```

### 签到 (Check-in)

客人到店确认的过程。

- **预约签到** - 有预约的客人到店
- **Walk-in** - 无预约直接到店的客人

### 结账 (Checkout)

服务完成后的付款流程。

支持的支付方式：
- 现金
- 信用卡/借记卡
- 礼品卡
- 会员积分
- 拆分支付（多种方式组合）

## 时间概念

### 营业时间 (Business Hours)

门店对外营业的时间段，按周设置。

### 排班 (Schedule)

员工的工作时间安排。

- **周模板** - 可复用的排班模板
- **实际排班** - 具体某一天的工作安排
- **调班** - 临时调整排班

### 日结 (Day-End Closeout)

每日营业结束后的对账流程。

<Callout type="tip">
日结是确保财务准确的重要流程，建议每天营业结束后完成。
</Callout>

## 营销概念

### 会员等级 (Membership Tier)

根据消费金额或积分划分的客户等级。

### 积分 (Points)

客户消费累积的奖励点数，可兑换折扣或礼品。

### 礼品卡 (Gift Card)

预付费卡，可用于支付服务费用。

### 标签 (Tag)

用于分类客户的标记，支持自动和手动标签。
